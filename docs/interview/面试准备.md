### 2. 简单介绍一下声动万全项目？

好的, 面试官, **声动万券**是一个类似于喜马拉雅听书的音频内容平台。 平台连接了创作者和消费者。 创作者可以发布专辑、上传音频并灵活配置售卖模式（整本或分集）；消费者则可以进行专辑的全站检索、音频的试听、订阅整本专辑或者是购买当前专辑的某些声音。 我们服务端结合腾讯云内容安全服务, 提供了专辑与声音的在线审核功能。除此之外，平台还基于 LangChain + LangGraph 搭建了一个智能助手，它能够查询数据库、进行网络搜索等。

该项目具体的技术栈和我负责的内容在简历上以及写了, 我就不过多说明了.


### 3. 简单介绍一下你的上段实习主要做了什么？

好的，面试官。上段实习我在医疗 MDT 项目组担任 Agent 工程师，核心工作就是**从零搭建 RAG 知识库的全链路**，主要分为**导入流水线**和**查询流程**两块：

**第一是导入流水线**。基于 LangGraph 的 StateGraph 设计了 6 个节点的自动化处理流程。用户上传 PDF 或 MD 文件后，自动经过解析、图片处理、文档切分、实体识别、向量化，最后入库 Milvus。因为是图架构，后续如果要支持 Word 或 Excel，只需要在图中新增对应节点就行，扩展性很好。

**第二是查询流程**。搭建了初步的 RAG 检索链路，采用了**三路召回策略**（向量检索 + HyDE 假设文档 + MCP 网络搜索）来提升短查询的召回率，再通过**两阶段排序**（RRF 融合打分 + BGE-Reranker 重排 + 动态截断）保证相关性，最终由大模型生成回答。
