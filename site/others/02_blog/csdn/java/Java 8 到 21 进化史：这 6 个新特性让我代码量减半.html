    Java 8 到 21 进化史：这 6 个新特性让我代码量减半

    很多人说 "Java 太啰嗦" 还停在 2014 年的 Java 8，实际上 Java 8 → 21 这十年，语言层面的进化可能是所有主流语言里最激进的。这篇文章不对 Java 说好话，只展示 6 个真正能提升效率的特性，每个都配传统写法和新写法的对比。

    一、你的代码真的跟上时代了吗？

    先看一段项目里真实存在的代码：

    // 一段 2025 年还在写的"Java 8 风格"代码
public class OrderDTO {
    private String orderId;
    private BigDecimal amount;

    // 手动 getter/setter/toString/equals/hashCode ...... 50 行
}

// 处理订单分类
String category;
switch (order.getType()) {
    case "VIP": category = "VIP订单";
        break;
    case "NORMAL": category = "普通订单";
        break;
    default: category = "未知";
}

// 构建 JSON 请求体
String body = "{\"orderId\":\"" + orderId + "\"," +
    "\"amount\":\"" + amount + "\"," +
    "\"user\":\"" + userId + "\"}";

// 空指针风险
if (order != null) {
    Address addr = order.getAddress();
    if (addr != null) {
        System.out.println(addr.getCity());
    }
}

    这段代码有什么问题？没有语法错误，能跑。但它写的每一个字，都在告诉别人"我还活在 Java 8 时代"。

    Java 8 是里程碑，但 Java 9~21 不是白更新的。有些特性学了之后真的回不去。

    二、核心概念：新特性到底在解决什么？

    把 Java 新特性按目标分 3 类，很好记：

    第一类：语法糖 —— 少打字 - var：编译器帮你写类型 - Switch 表达式：编译器帮你处理穿透 - Text Block：编译器帮你处理换行和缩进

    第二类：数据建模 —— 代码即文档 - record：编译器帮你写 POJO - 密封类：编译器帮你限制继承

    第三类：并发革命 —— IO 密集型场景的救星 - 虚拟线程：让操作系统帮你管理线程

    总结就是一句话：这些新特性不是在创造新概念，而是把"你手动做的事"交给编译器去做了。

    三、6 个核心特性详解（每个都配代码对比）

    特性 1：var —— 把重复的类型声明扔掉

    痛点：左右两边都写一遍类型，编译器明明知道是什么。

    // ❌ Java 8 写法：重复声明
Map<String, List<User>> userMap = new HashMap<String, List<User>>();
List<String> names = new ArrayList<String>();

// ✅ Java 10+ 写法：让编译器替你做
var userMap = new HashMap<String, List<User>>();
var names = new ArrayList<String>();
var entry = Map.entry("key", "value");

    收益：右值的类型越复杂，var 收益越大。尤其是嵌套泛型，少写 50% 的字符。

    限制：var 只能用于局部变量，不能用于字段、方法参数和返回值。而且必须有初始值。

    // ❌ 非法
var x;                // 没有初始化，无法推断
public var method() { return 42; }  // 不能作为返回值

// ✅ 合法
for (var entry : map.entrySet()) { ... }
try (var stream = Files.lines(Paths.get("file.txt"))) { ... }

    特性 2：Switch 表达式 —— 告别 break 穿透

    痛点：传统 switch 有 3 个大问题 —— 默认穿透（break 易忘）、不是表达式（无法赋值）、语法啰嗦。

    // ❌ 传统 switch：8 行代码，2 个 break
String result;
switch (day) {
    case MONDAY:
    case FRIDAY:
        result = "工作";
        break;
    case SUNDAY:
        result = "休息";
        break;
    default:
        result = "未知";
}

// ✅ Switch 表达式 + 箭头：3 行代码，0 个 break
String result = switch (day) {
    case MONDAY, FRIDAY -> "工作";
    case SUNDAY -> "休息";
    default -> "未知";
};

    箭头 -> 没有穿透问题，而且可以直接赋值给变量。如果逻辑复杂，用 yield 返回：

    String result = switch (score / 10) {
    case 10, 9 -> "优秀";
    case 8 -> "良好";
    case 6, 7 -> {
        // 多行逻辑
        log.warn("及格边缘");
        yield "及格";
    }
    default -> "不及格";
};

    特性 3：Text Block —— 再也不写 \n 拼接

    痛点：多行字符串（JSON、SQL、HTML）要么拼接 +，要么写 \n，可读性极差。

    // ❌ 传统：每一行都要拼接
String json = "{\n" +
    "  \"name\": \"Alice\",\n" +
    "  \"age\": 18\n" +
    "}";

// ✅ Text Block：三引号，所见即所得
String json = """
    {
      "name": "Alice",
      "age": 18
    }
    """;

    SQL 语句的效果更明显：

    // ❌ 每次看这段 SQL 都要在脑子里重新拼接
String sql = "SELECT u.id, u.name, o.amount " +
    "FROM users u " +
    "JOIN orders o ON u.id = o.user_id " +
    "WHERE u.age > 18 " +
    "ORDER BY o.amount DESC";

// ✅ 直接复制 SQL 到数据库工具就能跑
String sql = """
    SELECT u.id, u.name, o.amount
    FROM users u
    JOIN orders o ON u.id = o.user_id
    WHERE u.age > 18
    ORDER BY o.amount DESC
    """;

    特性 4：record —— 50 行 POJO 变 1 行

    痛点：Java 的 POJO 样板代码——每个类都要写 getter、setter、equals、hashCode、toString，加起来 50-100 行废话。

    // ❌ 传统 POJO：手写 50 行
public class User {
    private final String name;
    private final int age;

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
    public String getName() { return name; }
    public int getAge() { return age; }
    // equals / hashCode / toString 还要手写...
}

// ✅ record：1 行，以上全部自动生成
public record User(String name, int age) { }

    record 自动生成的内容： - 全参构造器 - name() 和 age() 访问器（注意不是 getName() / getAge()） - equals() 和 hashCode() - toString()

    // 使用
var user = new User("Alice", 18);
System.out.println(user.name());   // "Alice"
System.out.println(user);          // User[name=Alice, age=18]

    如果要做参数校验，用"紧凑构造器"：

    public record User(String name, int age) {
    public User {  // 编译器自动给你写好了 this.name = name; this.age = age;
        if (age < 0) {
            throw new IllegalArgumentException("年龄不能为负: " + age);
        }
    }
}

    特性 5：Pattern Matching —— instanceof + 强转一次搞定

    痛点：判断类型后还得强转，写了 20 年 Java 就重复了 20 年的废话。

    // ❌ 传统：instanceof + 强转
if (obj instanceof String) {
    String s = (String) obj;
    System.out.println(s.length());
}

// ✅ 模式匹配：一行解决
if (obj instanceof String s) {
    System.out.println(s.length());  // s 直接就是 String 类型
}

    Java 21 进一步升级到 switch 模式匹配，连 if-else 都省了：

    // Java 21：switch 直接匹配类型
public String describe(Object obj) {
    return switch (obj) {
        case null -> "null";
        case String s when s.length() > 5 -> "长字符串: " + s;
        case String s -> "短字符串: " + s;
        case Integer i when i < 0 -> "负数: " + i;
        case Integer i -> "正数: " + i;
        default -> "其他";
    };
}

    特性 6：虚拟线程 —— 百万并发不再是梦

    痛点：平台线程（操作系统线程）很贵——一个线程默认 1MB 栈内存，1 万个线程 JVM 就喘不过气了。所以大家用线程池、异步框架。

    // ❌ 传统：线程池限制并发数
ExecutorService executor = Executors.newFixedThreadPool(200);
// 如果有 2000 个 IO 请求，1800 个在排队

// ✅ 虚拟线程：不需要池，来多少接多少
ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();
// 可以同时处理 10 万个 IO 请求

    虚拟线程的核心原理是：执行 IO 操作时自动让出平台线程，IO 完成后自动恢复。对开发者来说，代码就是同步的、顺序的，但底层是异步的。

    // 虚拟线程 + 普通 for 循环 = 10 万并发
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 100_000; i++) {
        int taskId = i;
        executor.submit(() -> {
            // 同步写法，但底层是异步调度
            Thread.sleep(1000);  // 不阻塞平台线程
            System.out.println("任务 " + taskId + " 完成");
            return taskId;
        });
    }
}  // 等待所有任务完成

    线程模型演进：1:1（平台线程=OS线程）→ M:N（用户级线程）→ 虚拟线程（Java 21 直接交给 OS 调度）

    四、实战：用新旧两种方式实现订单分析

    下面用同一个需求，展示旧写法和新写法的差距。

    需求：有一个订单列表，需要过滤出金额 > 100 的订单，按用户分组，计算每个用户的订单总金额和平均金额，最后输出 JSON。

    旧写法（Java 8 风格）

    // 1. 定义订单类（手写 50 行）
public class Order {
    private String userId;
    private BigDecimal amount;

    public Order(String userId, BigDecimal amount) {
        this.userId = userId;
        this.amount = amount;
    }
    public String getUserId() { return userId; }
    public BigDecimal getAmount() { return amount; }
    // toString / equals / hashCode 手动写
}

// 2. 业务逻辑
public class OrderAnalyzer {
    public void analyze(List<Order> orders) {
        // 过滤
        List<Order> filtered = new ArrayList<>();
        for (Order o : orders) {
            if (o.getAmount().compareTo(BigDecimal.valueOf(100)) > 0) {
                filtered.add(o);
            }
        }

        // 分组统计
        Map<String, List<Order>> grouped = new HashMap<>();
        for (Order o : filtered) {
            String userId = o.getUserId();
            if (!grouped.containsKey(userId)) {
                grouped.put(userId, new ArrayList<>());
            }
            grouped.get(userId).add(o);
        }

        // 计算
        Map<String, String> result = new LinkedHashMap<>();
        for (Map.Entry<String, List<Order>> entry : grouped.entrySet()) {
            BigDecimal total = BigDecimal.ZERO;
            for (Order o : entry.getValue()) {
                total = total.add(o.getAmount());
            }
            BigDecimal avg = total.divide(
                BigDecimal.valueOf(entry.getValue().size()),
                2, RoundingMode.HALF_UP);
            result.put(entry.getKey(),
                "total=" + total + ", avg=" + avg);
        }

        System.out.println(result);
    }
}

    约 60 行代码，3 层 for 循环嵌套，可读性差，修改困难。

    新写法（Java 17+ 风格）

    // 1. 订单类：1 行，自动生成构造器、equals、hashCode、toString
public record Order(String userId, BigDecimal amount) { }

// 2. 业务逻辑
public class OrderAnalyzer {
    public void analyze(List<Order> orders) {
        var result = orders.stream()
            .filter(o -> o.amount().compareTo(BigDecimal.valueOf(100)) > 0)
            .collect(Collectors.groupingBy(
                Order::userId,
                Collectors.collectingAndThen(
                    Collectors.toList(),
                    list -> {
                        var total = list.stream()
                            .map(Order::amount)
                            .reduce(BigDecimal.ZERO, BigDecimal::add);
                        var avg = total.divide(
                            BigDecimal.valueOf(list.size()),
                            2, RoundingMode.HALF_UP);
                        return "total=" + total + ", avg=" + avg;
                    }
                )
            ));

        System.out.println(result);
    }
}

    约 20 行代码，没有 for 循环，没有 if 判断，每个步骤一眼能看出来在做什么。

    差距总结

    维度 旧写法 新写法
    代码量 ~60 行 ~20 行
    临时变量 8 个 1 个
    修改难度 改一处要翻 3 个 for 循环 改一个 filter / map
    可读性 需要读完整段才知道意图 链式调用，意图一目了然

    五、避坑指南

    var 的 3 个坑

    // 坑 1：必须初始化，否则推断不了
var x;  // 编译错误

// 坑 2：不能用于字段
public class User {
    private var name;  // 编译错误
}

// 坑 3：多变量声明不行
var a = 1, b = 2;  // 编译错误

    record 的 3 个限制

    // 限制 1：不可变。没有 setter，不能修改字段
var user = new User("Alice", 18);
// user.setName("Bob");  // setter 不存在

// 限制 2：不能被继承（隐式 final）
public class VipUser extends User { }  // 编译错误

// 限制 3：不能添加实例字段
public record User(String name) {
    private int id;  // 编译错误，只能声明静态字段
}

    虚拟线程的 4 个注意点

    // 注意 1：不要池化虚拟线程
// ❌
var executor = Executors.newFixedThreadPool(1000);
// ✅ 每次创建的开销极低，池化反而浪费
var executor = Executors.newVirtualThreadPerTaskExecutor();

// 注意 2：不要用 synchronized 锁（会导致固定 pinning）
// ❌
synchronized (lock) { ... }
// ✅ 用 ReentrantLock 替代
lock.lock();
try { ... } finally { lock.unlock(); }

// 注意 3：ThreadLocal 谨慎用（虚拟线程数多，内存放大）
// 注意 4：不是所有场景都适合——计算密集型任务用虚拟线程没意义

    升级路线建议

    Java 8 --→ Java 17 --→ Java 21
  LTS         LTS          LTS
  (EOL)     (当前)       (推荐)
    你现在的版本 建议目标 理由
    Java 8 Java 17 跨过 9~16，直接进入现代 Java
    Java 11 Java 21 11 和 17 差距不大，直接到 21 吃虚拟线程红利
    Java 17 按需 17 足够用，21 的虚拟线程是主要动力

    六、总结：一张表记住

    特性速查

    特性 版本 一句话价值 学它要多久
    var Java 10 少写重复类型声明 5 分钟
    Switch 表达式 Java 14→17 干掉 break 和穿透 10 分钟
    Text Block Java 13→15 多行字符串不再拼接 5 分钟
    record Java 14→16 50 行 POJO → 1 行 10 分钟
    Pattern Matching Java 16→21 instanceof + 强转一次搞定 10 分钟
    虚拟线程 Java 21 10 万并发 = 写普通 for 循环 30 分钟

    一句话记住这 6 个特性

    var 省类型，switch 省 break，Text Block 省拼接，record 省 POJO，模式匹配省强转，虚拟线程省线程。
